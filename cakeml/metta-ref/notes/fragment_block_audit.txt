CakeML MeTTa marked fragment block audit
========================================

Scope
-----

This audits the functions between:

- BEGIN GENERATED PRE_EVAL_FRAGMENTS
- END GENERATED PRE_EVAL_FRAGMENTS
- BEGIN GENERATED POST_EVAL_FRAGMENTS
- END GENERATED POST_EVAL_FRAGMENTS

in:

- sml/metta_m1.sml
- cml/metta_m1.cml

Classification vocabulary
-------------------------

- translated helper: generated by the CakeML translator from a HOL definition,
  with a translator theorem and app spec in
  hol/metta_m1_cake_bridgeScript.sml.
- CF wrapper: handwritten CakeML quoted in HOL and proved with CF tactics
  against a HOL specification.
- exploratory only: handwritten SML/CakeML mirror code that is regression
  tested and linted, but not itself connected to HOL semantics by a CakeML
  translator theorem or CF proof.

Marked SML/CakeML block classification
--------------------------------------

eval_return_fragment:
  classification: exploratory only
  reason: HOL has eval_return_fragment_agrees_with_eval_m1_rec, export
  roundtrip theorems, a CakeML translator certificate for the numeric HOL
  eval_return_fragment, and a CF wrapper theorem for
  bridge_eval_return_fragment_hand. The bridge theory also proves that the
  bridge_surface_atom-level return core imports to the numeric
  eval_return_fragment result, and CF-proves the tiny string-boundary predicate
  bridge_surface_is_return_symbol_hand. The marked SML/CakeML body is now
  factored into eval_return_fragment_core_is_return_head,
  eval_return_fragment_core_items, and the public eval_return_fragment_core,
  matching the proved factored bridge target. The bridge theory now also
  relates a source-string atom model to bridge_surface_atom and proves that the
  source-level return core refines the surface-level return core. The exact
  marked prototype return-core block is quoted in the HOL bridge theory, but
  its direct CF proof still has unresolved raw pattern-match obligations. The
  marked block remains exploratory until the actual shipped string-world block
  is CF-proved against the relation or replaced by a generated/translated
  artifact.

eval_add_fragment:
  classification: exploratory only
  reason: the bridge theory has translated/CF-backed primitive values helpers
  for evaluated +, <, ==, and/or/not, but the marked eval_add_fragment body
  calls the handwritten evaluator and is not itself CF-proved.

eval_eval_fragment:
  classification: exploratory only
  reason: HOL has an eval branch fragment theorem and export roundtrip, but the
  marked SML/CakeML body is handwritten and not certified.

eval_chain_fragment:
  classification: exploratory only
  reason: HOL has eval_chain_fragment_agrees_with_eval_m1_rec, and the bridge
  now has translated chain payload helpers for substitution and result
  concatenation, but the marked SML/CakeML chain fragment still calls the
  handwritten evaluator and is not itself certified.

eval_case_fragment:
  classification: exploratory only
  reason: HOL has a case branch fragment theorem and export roundtrip, but the
  marked SML/CakeML body is handwritten and not certified.

m1_typed_eval:
  classification: exploratory only
  reason: this is a local handwritten helper in the marked block, not a HOL
  fragment and not certified.

eval_typed_fragment:
  classification: exploratory only
  reason: HOL has typed fragment theorems, but the marked SML/CakeML body
  delegates to handwritten m1_typed_eval.

eval_evalc_fragment:
  classification: exploratory only
  reason: HOL has evalc fragment theorems, but the marked SML/CakeML body uses
  handwritten runner/type helpers and is not certified.

eval_vec_cons_fragment:
  classification: exploratory only
  reason: HOL has Vec/Cons fragment theorems, but the marked SML/CakeML body
  calls a handwritten dependent-type helper and is not certified.

Current non-claim
-----------------

None of the marked SML/CakeML fragment blocks is currently a verified CakeML
implementation of the corresponding HOL evaluator fragment. The marked blocks
are useful executable witnesses and regression targets. The proof-bearing path
is the separate bridge theory: translated pure helpers first, then CF wrappers
or stateful/FFI proofs where appropriate.
